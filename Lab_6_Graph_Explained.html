Graph Concepts You Need to Know

Graph Types: - Directed Graph: edges have a direction (A → B is not the same as B → A) - Undirected Graph: edges have no direction (A—B is same as B—A) Representations: - Adjacency Matrix: 2D array [][] where matrix[i][j] = weight of edge i→j - Adjacency List: Array of lists where list[i] contains all neighbors of i Key Algorithms: - DFS (Depth-First Search): explores as far as possible along each branch before backtracking. Uses a stack (or recursion). - BFS (Breadth-First Search): explores all neighbors at current depth first. Uses a queue. - Dijkstra's Algorithm: finds the shortest path from a source to all other nodes in a weighted graph.
ConceptDefinitionUsed In
Vertex (Node)A point in the graphAll 3 programs
EdgeA connection between two verticesAll 3 programs
Adjacency ListEach vertex stores a list of its neighborsComputer Connection
Adjacency Matrix2D array where [i][j] = edge weight between i and jCameron, Simple Graph
DFS (Recursive)Depth-First Search using recursionComputer Connection

Key Insight: The choice between adjacency matrix and adjacency list depends on the graph density. A matrix uses more memory (V²) but gives O(1) edge lookup. A list uses less memory for sparse graphs but takes O(V) to check if an edge exists.

Behind the Scenes — DFS Execution Trace

Input: vertices=4, edges=3: (1,2), (2,3), (3,4), query: isConnected(1,4)

Step 1: Build the Adjacency List

After adding all edges (undirected), the graph looks like:

VertexNeighbors
1[2]
2[1, 3]
3[2, 4]
4[3]

Each edge is added in both directions (undirected). For example, addEdge(1,2) adds 2 to adj[1] AND 1 to adj[2].

Step 2: DFS Recursion Trace for isConnected(1, 4)

dfs(current=1, dest=4)

Actionvisited[] (1-indexed)Result
v[1]v[2]v[3]v[4]
current=1 ≠ 4, mark visited[1]=true TFFF → check neighbors [2]
neighbor 2 is unvisited → recurse TTFF → dfs(2,4)
current=2 ≠ 4, mark visited[2]=true
neighbors [1,3]: skip 1 (visited), go to 3		 TTTF → dfs(3,4)
current=3 ≠ 4, mark visited[3]=true
neighbors [2,4]: skip 2 (visited), go to 4		 TTTT → dfs(4,4)
current=4 == dest=4 → PATH FOUND! TTTT return true

Step 3: Call Stack Unwinding

When dfs(4,4) returns true, each recursive call passes it back up:

dfs(4,4) → true (base case: current == dest)
↑ dfs(3,4) got true from neighbor 4 → return true
↑ dfs(2,4) got true from neighbor 3 → return true
↑ dfs(1,4) got true from neighbor 2 → return true
↑ isConnected(1,4) → prints "Connected"

Final Answer

Path exists: 1 → 2 → 3 → 4. Computers 1 and 4 are connected!

Exam Tip — Dijkstra's Algorithm Summary:
  1. Initialization: Set distance[source] = 0, all others = ∞ (MAX_VALUE). Add source to unsettled.
  2. Extract min: Pick the unsettled vertex with smallest distance (greedy choice). This is now "settled" — its distance is final.
  3. Relax neighbors: For each unsettled neighbor, check if going through the settled node is shorter. If yes, update distance and add to unsettled.
  4. Repeat steps 2-3 until all nodes are settled (or destination is reached).
  5. Requirement: Edge weights must be non-negative.
  6. Complexity: O(V²) with this simple implementation, O(E log V) with a priority queue.

DFS vs Dijkstra — When to use which:
  • DFS = Use when you only need to know IF a path exists (unweighted graph). Simple recursion, O(V+E).
  • Dijkstra = Use when you need the SHORTEST path in a weighted graph. More complex, O(V²) or O(E log V).
  • BFS = Use for shortest path in an UNWEIGHTED graph (not shown here, but know it for the exam!).
