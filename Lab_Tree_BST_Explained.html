What is a Binary Search Tree (BST)?

A BST is a tree data structure where each node has at most 2 children (left and right), and they follow a special ordering rule:

  • Left child → contains values smaller than the parent
  • Right child → contains values larger than the parent
  • No duplicate values (in these labs)
50 / \ 30 70 / \ \ 20 40 80

This ordering makes searching very fast — at each node, you decide to go left or right, eliminating half the remaining tree. This is O(log n) on average.

Pre-order Traversal

Root → Left → Right

Visit node FIRST, then left subtree, then right subtree.

Used for: copying the tree

In-order Traversal

Left → Root → Right

Visit left subtree FIRST, then node, then right subtree.

Gives nodes in sorted ascending order!

Post-order Traversal

Left → Right → Root

Visit left subtree, then right subtree, then node LAST.

Used for: deleting the tree

Key Insight: All BST operations (insert, search, delete) use the same pattern: start at root, compare the value, go left if smaller, go right if larger, recurse until you find null (insert here) or the value (search found it). This is called a recursive approach — a method that calls itself.

BST Rules Summary for Exam:
  1. Each node has at most 2 children (left and right)
  2. Left subtree contains values smaller than the parent
  3. Right subtree contains values larger than the parent
  4. No duplicates (varies by implementation — some warn, some silently ignore)
  5. In-order traversal (Left → Root → Right) gives values in sorted order
  6. Search, insert, delete are O(log n) on average (O(n) worst case)
